Internet Technologies - Ex04 test guide
In our tests we have several different suites, each suite represents a different server setup, that is different handler are loaded on the server, The following list will describe what is the characterization of each suite, and which tests are tested on that setup.

  • Suite 1
    • Basic static test
    • Basic static non existing file test
    • Basic static non http format test
    • Basic static head test
    • Basic cookie test
    • Stop server test

    Suite 2
    • Post server and a get request test
    • Stop server test

    Suite 3
    • Get server and a post request test
    • Stop server test

    Suite 4
    • A basic test for URI query
    • A basic test for body query
    • Stop server test
The tests are launched sequentially (mainly that a server wouldn't stop before all the requests were handled (although the server does reply to all currently handled request), when the tests are concurrent, many different issues may arise (mainly because of the shutdown check). In the test we though it will be more logical to test each function as on its own, and not mix several tests apart (this will generally should provide consistency in the test results).

Suites

  • Suite 1

    This suite tests the basic static server handler and the cookie handler, both loaded on the '/root' resource

  • Suite 2

    This suite tests the basic static server handler when loaded only for POST requests, the handler is loaded with the '/root' resource.

  • Suite 3

    This suite tests the basic static server handler when loaded only for GET requests, the handler is loaded with the '/root' resource.

  • Suite 4

    This suite test our own extra handler for records (mainly for access to json files and queries), the handler is loaded with the '/root' resource.

Tests

  • Basic static test

    This basic test tests, if a proper response is replied upon a valid file request.

  • Basic static non existing file test

    A basic test for a non existing file, a status code of 404 should be returned.

  • Basic static non http format test

    This test test how is a non http format request handled. The answer should contain 500 code.

  • Basic static head test

    This test sends a basic head request, and assert that only the heads were returned, and they are the same as the get request for the same file.

  • Basic cookie test

    This test sends a request containing a cookie, and expects to receive this cookie back in the response.

  • Stop server test

    This test will stop the server, and than assert the server has been stopped.

  • Post server and a get request test

    This test loads up a static server for POST requests and try to approach this server through a GET request

  • Get server and a post request test

    This test loads up a static server for GET requests and try to approach this server through a POST request

  • A basic test for getting a record through a URI query

    This basic test, tests if a query is received through the path, whether a response containing the appropriate object from the supplied by path json file.

  • A basic test for getting a record through a body

    This basic test, tests if a query is received through the body, whether a response containing the appropriate object from the supplied by path json file.
